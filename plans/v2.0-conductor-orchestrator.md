## Vision (shopper vs system)

| Layer | Visible? | Role |
|-------|----------|------|
| **Conductor** | **No** | System admin — decides next action each tick |
| **Social agent** | Yes (voice) | Writes shopper-facing lines from conductor briefs |
| **Intent / topic-guard** | No | Scope filter when conductor requests |
| **RAG + logic + synthesis** | No | Catalog pipeline when conductor starts it |

Shopper sees only **zooplus Assistant** bubbles + typing — same v1.4 UX shell.

---

## Conductor decision loop (each ~5s or on event)

```json
{
  "action": "emit_message | start_catalog | wait | complete",
  "message_brief": "one NEW idea for social agent (never repeat prior lines)",
  "wait_seconds": 5,
  "reason": "internal"
}
```

**Rules encoded in prompt:**
- Say shop scope (dog/cat only) **once** in the whole turn
- Later ticks = progress only (searching, narrowing, almost ready)
- `start_catalog` only if lane is catalog_search and not yet running
- `complete` when catalog receipt ready or social-only turn done

**Model:** `opencode-go/minimax-m2.7` (stronger reasoning; was m2.5 in v1.4)
