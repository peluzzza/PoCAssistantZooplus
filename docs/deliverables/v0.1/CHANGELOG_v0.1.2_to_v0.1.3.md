# Changelog ‚Äî from PPT deck v0.1.2 to current `releases`

---

## 1. Architecture and routing (agentic)

| Area | Before (v0.1.2 PPT) | Now |
|------|---------------------|-----|
| Intent | Intent agent ‚Üí lane on every turn | **Conductor-first** (`ZOOPLUS_CONDUCTOR_INTENT=1`): classifies topic **before** RAG |
| Greetings / social | Could trigger catalog search (slow) | Fast-path social + conductor: **no RAG** on *hello*, *thanks*, help |
| Intent timeout | Slow full cascade | Topic fallback + single-agent intent; parallel catalog prefetch |
| Multilingual | Mostly English hints | **OpenCode interprets any language**; catalog lexicon in prompt |
| Fixed keywords | Hints with dog/cat/perro/gato | **Removed**; vocabulary from `routing_lexicon.json` (ingest) |
| LLM repair | ‚Äî | Opt-in `ZOOPLUS_INTENT_REPAIR=0` (conductor decides; repair only with catalog signal) |

**Key files:** `src/agents/intent_agent.py`, `src/lanes/orchestrator.py`, `src/rag/catalog_lexicon.py`, `src/agents/prompts.py`

## 2. Conversational UX (Chat UI)

## 5. Performance and cache

- **In-process TTL cache:** intent, chat, retrieval (`ZOOPLUS_CACHE=1`).
- **Optional Redis** (`ZOOPLUS_CACHE_BACKEND=redis`, `ZOOPLUS_REDIS_URL`): cache mirror + pre-indexed lexicon.
- **Lexicon at ingest:** `artifacts/index/routing_lexicon.json` (brands, tokens, pet_types from JSON).
- EUR price: extended parser (`entre X y Y`, Spanish).
- Faster F2 gates; agentic/social tests in F3.
