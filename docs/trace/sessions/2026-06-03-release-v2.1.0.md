# Release v2.1.0 — Python 3.11 + dependency alignment

| Tag | `v2.1.0` |

## Changes

- `requires-python = ">=3.11,<3.13"` (blocks accidental 3.13 as supported target)
- `.python-version` → 3.11.9
- Pinned dependency ranges in `pyproject.toml` + `requirements-lock.txt`
- `docs/DEPENDENCIES.md`, `scripts/resolve_python.py`
- Quality gates warn when local Python ≠ 3.11
- CI: explicit Python 3.11 verify step

## Note

Local Windows may still run gates on 3.13 with WARNING; **CI/Docker remain 3.11**.
